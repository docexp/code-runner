# Continuous Integration

## Context & Goals

The repo has one existing workflow (`auto-github-pr.yml`) that opens pull requests but no workflow that runs tests or type-checks on those PRs. This spec introduces a full CI pipeline that validates every PR to `next` and every push to `next` and `main`. It also updates the existing auto-PR workflow to target the `next` branch (aligned with the branch model defined in spec 006-000).

## Scope

**In scope:**
- `.github/workflows/ci.yml` — main CI workflow (typecheck, unit tests, e2e)
- Update `.github/workflows/auto-github-pr.yml` — change `target_branch` from `main` to `next`
- Nx affected detection for PRs to skip unchanged packages
- Playwright e2e running in CI mode via the existing `react-e2e` configuration

**Out of scope:**
- Build artefact publishing or npm release (spec 006-000)
- Performance benchmarks
- Visual regression testing
- Matrix testing across multiple Node/Bun versions (CI targets Node 22 only, matching `.prototools`)

## Package Impact

No package code changes. New and modified files in `.github/workflows/`.

## Requirements

### Functional Requirements

- FR-001: The CI workflow must trigger on `pull_request` events targeting `next` or `main`, and on `push` events to `next` or `main`.
- FR-002: On pull requests, only **affected** projects must be type-checked and unit-tested (using `nx affected`) to keep CI fast.
- FR-003: On push to `next` or `main`, **all** projects must be type-checked and unit-tested (using `nx run-many`).
- FR-004: E2e tests must run in every workflow invocation (PRs and pushes) against Chromium only.
- FR-005: Unit tests and e2e tests must run as separate jobs so they can fail independently and are visible as distinct status checks.
- FR-006: The workflow must use Node 22 and Bun (latest stable) to match the developer environment.
- FR-007: `bun install` must use `--frozen-lockfile` to guarantee reproducible installs.
- FR-008: The `auto-github-pr.yml` workflow must target `next` instead of `main`.

### Non-Functional Requirements

- NFR-001: All jobs must use `ubuntu-latest`.
- NFR-002: `node_modules` and `.nx/cache` should be cached using `actions/cache` to reduce job duration.
- NFR-003: No secrets stored in plain text; the only secret needed is `GH_PAT` (already present) for the auto-PR action.
- NFR-004: The e2e job must depend on the build of all packages completing (via `build-deps` in the `react-e2e` Nx target) before Playwright launches.

## Technical Design

### Workflow trigger

```yaml
on:
  push:
    branches: [next, main]
  pull_request:
    branches: [next, main]
```

### Jobs overview

```
ci.yml
├── typecheck          (nx affected / nx run-many)
├── unit-test          (nx affected / nx run-many)
└── e2e                (nx run react-e2e:e2e, always full)
    └── depends on: unit-test
```

`e2e` depends on `unit-test` to avoid running e2e when unit tests already fail, and to ensure built artefacts are available on the same runner.

> Note: `e2e` rebuilds packages on its own runner because GitHub Actions jobs run in isolated environments. The `dependsOn: ["build-deps"]` inside the Nx `react-e2e` target handles this automatically when `nx run react-e2e:e2e` is invoked.

### Affected vs run-many logic

Both `typecheck` and `unit-test` share the same pattern:

```yaml
- name: Run (PR — affected only)
  if: github.event_name == 'pull_request'
  run: bunx nx affected -t <target> --base=origin/${{ github.base_ref }} --head=HEAD --output-style=stream

- name: Run (push — all projects)
  if: github.event_name == 'push'
  run: bunx nx run-many -t <target> --output-style=stream
```

### Node + Bun setup (shared across all jobs)

```yaml
- uses: actions/checkout@v4
  with:
    fetch-depth: 0   # required for nx affected to compute the diff

- uses: actions/setup-node@v4
  with:
    node-version: 22

- uses: oven-sh/setup-bun@v2

- uses: actions/cache@v4
  with:
    path: |
      node_modules
      .nx/cache
    key: ${{ runner.os }}-bun-${{ hashFiles('bun.lock') }}
    restore-keys: ${{ runner.os }}-bun-

- run: bun install --frozen-lockfile
```

> `fetch-depth: 0` is required so Nx can compare the current HEAD against the base branch to determine affected projects.

### Typecheck job

```yaml
typecheck:
  runs-on: ubuntu-latest
  steps:
    - <setup steps>
    - name: Typecheck (PR — affected only)
      if: github.event_name == 'pull_request'
      run: bunx nx affected -t typecheck --base=origin/${{ github.base_ref }} --head=HEAD --output-style=stream

    - name: Typecheck (push — all)
      if: github.event_name == 'push'
      run: bunx nx run-many -t typecheck --output-style=stream
```

### Unit-test job

```yaml
unit-test:
  runs-on: ubuntu-latest
  steps:
    - <setup steps>
    - name: Test (PR — affected only)
      if: github.event_name == 'pull_request'
      run: bunx nx affected -t test --base=origin/${{ github.base_ref }} --head=HEAD --output-style=stream

    - name: Test (push — all)
      if: github.event_name == 'push'
      run: bunx nx run-many -t test --output-style=stream
```

### E2e job

The `react-e2e` Playwright config handles vite dev server start automatically (`webServer` stanza) when `CI=true`. The `reuseExistingServer: !process.env['CI']` setting means CI always starts a fresh server.

```yaml
e2e:
  runs-on: ubuntu-latest
  needs: unit-test
  steps:
    - <setup steps>
    - name: Install Playwright browsers
      run: bunx playwright install --with-deps chromium

    - name: E2e
      run: bunx nx run react-e2e:e2e --output-style=stream
      env:
        CI: true
```

> Only Chromium is installed. The `playwright.config.ts` already limits projects to `chromium`, so `--with-deps chromium` avoids downloading Firefox and WebKit (~300 MB each).

### Updated `auto-github-pr.yml`

Change `target_branch: main` → `target_branch: next`.

### Full `ci.yml` skeleton

```yaml
name: CI

on:
  push:
    branches: [next, main]
  pull_request:
    branches: [next, main]

jobs:
  typecheck:
    name: Type-check
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: actions/setup-node@v4
        with:
          node-version: 22
      - uses: oven-sh/setup-bun@v2
      - uses: actions/cache@v4
        with:
          path: |
            node_modules
            .nx/cache
          key: ${{ runner.os }}-bun-${{ hashFiles('bun.lock') }}
          restore-keys: ${{ runner.os }}-bun-
      - run: bun install --frozen-lockfile
      - name: Typecheck (PR)
        if: github.event_name == 'pull_request'
        run: bunx nx affected -t typecheck --base=origin/${{ github.base_ref }} --head=HEAD --output-style=stream
      - name: Typecheck (push)
        if: github.event_name == 'push'
        run: bunx nx run-many -t typecheck --output-style=stream

  unit-test:
    name: Unit tests
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: actions/setup-node@v4
        with:
          node-version: 22
      - uses: oven-sh/setup-bun@v2
      - uses: actions/cache@v4
        with:
          path: |
            node_modules
            .nx/cache
          key: ${{ runner.os }}-bun-${{ hashFiles('bun.lock') }}
          restore-keys: ${{ runner.os }}-bun-
      - run: bun install --frozen-lockfile
      - name: Test (PR)
        if: github.event_name == 'pull_request'
        run: bunx nx affected -t test --base=origin/${{ github.base_ref }} --head=HEAD --output-style=stream
      - name: Test (push)
        if: github.event_name == 'push'
        run: bunx nx run-many -t test --output-style=stream

  e2e:
    name: E2e tests
    runs-on: ubuntu-latest
    needs: unit-test
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: actions/setup-node@v4
        with:
          node-version: 22
      - uses: oven-sh/setup-bun@v2
      - uses: actions/cache@v4
        with:
          path: |
            node_modules
            .nx/cache
          key: ${{ runner.os }}-bun-${{ hashFiles('bun.lock') }}
          restore-keys: ${{ runner.os }}-bun-
      - run: bun install --frozen-lockfile
      - name: Install Playwright browsers
        run: bunx playwright install --with-deps chromium
      - name: E2e
        run: bunx nx run react-e2e:e2e --output-style=stream
        env:
          CI: true
```

## Nx Considerations

- `fetch-depth: 0` is mandatory for `nx affected` to work; the default shallow clone (`fetch-depth: 1`) gives Nx no ancestry to diff against.
- `.nx/cache` is safe to cache across runs because Nx cache keys are content-addressed.
- No new Nx targets are needed — `typecheck`, `test`, and `e2e` are all already inferred by the existing plugins in `nx.json`.

## Security Considerations

- No secrets are used in `ci.yml`. All tests run without authentication.
- `GH_PAT` is used only in `auto-github-pr.yml` and already exists as a repository secret. The Developer must not reference it in `ci.yml`.
- The `GITHUB_TOKEN` permissions are not overridden in `ci.yml`; default read-only permissions are sufficient.
- Cache keys are based on `bun.lock` content hash, preventing cache poisoning from dependency changes.

## Implementation Notes for Developer

1. Create `.github/workflows/ci.yml` with the skeleton above (expand the `<setup steps>` placeholder with the full setup block shown in the "Node + Bun setup" section).
2. Edit `.github/workflows/auto-github-pr.yml`: change `target_branch: main` to `target_branch: next`.
3. Verify the lock file name is `bun.lock` (not `bun.lockb`) — check with `ls bun.lock*` at the repo root. Adjust the `hashFiles(...)` path if different.
4. Verify with: `bunx nx run-many -t build --output-style=stream` (build must pass before the patch is merged).
5. After merging, confirm that a test PR to `next` triggers the workflow and the matrix of three jobs appears in the GitHub Actions tab.

## Acceptance Criteria

- [ ] `ci.yml` exists in `.github/workflows/` and is valid YAML.
- [ ] A PR to `next` triggers `typecheck`, `unit-test`, and `e2e` jobs.
- [ ] A push to `next` or `main` triggers all three jobs against the full project graph.
- [ ] `unit-test` running on a PR only executes tests for affected packages (verified by inspecting logs when only one package is touched).
- [ ] `e2e` job only starts after `unit-test` succeeds.
- [ ] `auto-github-pr.yml` creates PRs targeting `next` instead of `main`.
- [ ] Build passes: `bunx nx run-many -t build --output-style=stream` reports no errors.
